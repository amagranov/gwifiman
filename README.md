# gwifiman
Реализует возможность автоматической смены пароля для гостевой WiFi сети на роутерах с прошивкой OpenWRT.<br>

# Для чего использовать?
Часто курьеры, осуществляющие доставку "просят WiFi". Необходимость каждому рассказывать название сети, пароль помноженная на старческую параною относительно того, что курьер может "поделиться" твоим WiFi и что со временем полгорода будет "сидеть" в твоей домашней сети родила данный проект.<br>

# Идея
Нужен механизм, который будет периодически менять пароль на гостевом WiFi и способ быстрого подключения для гостя.<br>
Способом подключения был выбран QR код, отображаемый на экране собственного смартфона, который можно предъявить гостю.<br>

# Как работает?
Если у вас есть в распоряжении какой-нибудь веб-сервер на NAS или хостинге, то можно реализовать следующую схему:<br>
1. На роутере выполняется скрипт, который производит смену пароля для WiFi сети, затем отправляет данные при помощи POST запроса на ваш веб сервер.<br>
2. Сервер сохраняет актуальные данные в json или БД. Разумеется необходимо организовать безопасность передачи и получения таких запросов, но это уже "дело техники".<br>
3. По запросу сервер возвращает страницу с QR кодом для подключения. Здесь также нужно организовать какую-то простую схему проверки полномочий доступа к этой странице. Например, если у вас фиксированный домашний IP можно выдавать страницу только по запросу с этого IP.<br>
Но это все хорошо, если у вас есть такой сервер. А что делать, если его нет?<br>

На роутере есть встроенный uHTTPd http сервер, который позволяет хостить на нем свой "сайт". Пользуясь этой возможностью я решил разместить на нем страницу, которая будет отображать QR код для подключения к гостевой WiFi сети.<br>
Данные для отображения QR кода будут храниться в javascript, который будет обновляться вместе со сменой пароля. А сам javascript мы разместим в ОЗУ роутера, чтобы минимизировать количество записей во flash память устройства.<br>
Cron будет запускать shell скрипт, который будет создавать новый пароль для этого WiFi, изменять его настройки на роутере и заменять javascript, отображающий QR код на любом устройсте, которое уже подключено к домашней сети.<br>

# Установка
Если еще нет настроенной гостевой WiFi сети, настраиваем ее по одному из гайдов. Например:<br>
[тут](https://openwrt.org/docs/guide-user/network/wifi/guestwifi/configuration_webinterface)<br>
[тут](https://lsetc.ru/gostevoj-wi-fi-na-openwrt-cherez-luci/)<br>
[или тут](https://developersu.blogspot.com/2015/08/wifi-openwrt-luci.html)<br>

Затем подключаемся к роутеру по SSH.<br>
Качаем скрипт install.sh, даем разрешения на запуск и запускаем скрипт установки<br>
```
curl -O https://raw.githubusercontent.com/amagranov/gwifiman/main/install.sh
chmod +x install.sh
./install.sh
```
В процессе установки скрипт запрашивает какую сеть из настроенных на роутере использовать в качестве гостевой, какое шифрование у этой сети (обычно WPA), является ли сеть скрытой или отображается в списке сетей (я использую скрытую) и через какой интервал времени в минутах производить смену пароля.<br>
После окончания установки для отображения QR кода нужно открыть в браузере на устройстве, подключенном к домашней сети ссылку http://{IP адрес роутера}/guest.html<br>
По ссылке отображается QR код и обратный отсчет со временем действия текущего пароля.<br>
После первой установки или перезагрузки роутера обратный отсчет может не соответствовать реальному времени действия пароля, т.к. смена производится по расписанию через cron job.<br>
